eLua Project News

18 de Janeiro de 2010

Lançada a versão 0.7 de eLua. As principais características adicionadas estão descritas a seguir. Para uma lista mais completa das novidades, por favor consulte nossa página de Histórico das Versões:

  • Suporte oficial à eLua em Windows, com novo tutorial (por enquanto apenas em inglês) aqui.
  • Suporte ao kit PUC-Rio eLua Demo Board, com MCU NXP LPC2468.
  • Suporte ao kit Olimex STR-E912, com MCU ST Electronics STR912FW44X.
  • Novo FAT File System para SD/MMC Cards, genérico e portátil. Mais detalhes aqui.
  • Suporte ao patch Lua EGC (Emergency Garbage Collection). Mais detalhes aqui.
  • Suporte à LuaRPC. Remote Procedure Calls no mundo embedded (!!!).
  • Novo comando para o shell de eLua. Lista o conteúdo de arquivos de texto.
  • Novos modos de build do ROM File System: Compress e Compile. Mais detalhes aqui.
  • Mais conteúdo traduzido para Português.
  • Novas funcionalidades do Módulo ADC (conversor analógico-digital) e sua doc melhorada.
  • Mais tutoriais e melhorias gerais na documentação e no site.

01 de Novembro de 2008

A versão 0.5 foi lançada! Dentre outras novidades, essa versão vem com suporte TCP/IP para eLua (!!!!). Segue a lista das atualizações:

  • Suporte para microcontroladores STR7 da ST Microeletronics
  • Suporte TCP/IP usando o stack uIP
  • Suporte do console e do *shell* sobre TCP/IP além da conexão serial.
  • Módulo "net" (interface **eLua** para as funções TCP/IP)
  • Módulo "cpu" (interface **eLua** para o dados do microcontrolador escolhido)
  • Novos exemplos: morse.lua (código morse), lhttpd.lua (script Lua para servidor HTTP)
  • Cross Compiling Lua (compila no computador, roda na plataforma)
  • XMODEM pode agora receber o *bytecode*, além de código fonte Lua
  • Um *buffer* dinâmico para o XMODEM (cresce se necessário) ao invés de tamanho fixo
  • Documentação atualizada do projeto

Adicionado também um novo tutorial sobre como usar eLua com o microcontrolador STR7 da ST. O restante da página do projeto foi atualizada refletindo o Status atual do projeto (isso fica mais evidente nas páginas de Perguntas mais frequentes, Exemplos e Status.

Importante: não é necessário atualizar o binutils para a versão 2.19 para os microcontroladores Cortex. O tutorial do Cortex GCC para Cortex já foi atualizado com esta informação.

Desfrute dessa nova atualização. A próxima versão será focada na redução das memórias Flash e RAM de eLua, além de muitas outras surpresas :)

10 de Setembro de 2008

Foi liberada a versão 0.4.1! É uma atualização de pouca importância, pois seu propósito principal é se manter atualizada com Lua, dessa forma, eLua está agora com a mais recente versão de Lua (5.1.4). Provavelmente você não precisa fazer a atualização para esta versão (já que a 5.1.4 corrige alguns poucos e exóticos problemas da 5.1.3), logo, estou disponibilizando somente os fontes, sem nenhum executável. Segue abaixo o log das alterações:

  • Alterada a estrutura do sistema de arquivos; agora você pode gerar executáveis de ambas as versões de Lua (ponto flutuante e somente inteiro) a partir do mesmo diretório;
  • Feita uma biblioteca matemática configurável usando-se o próprio mecanismo de bibliotecas já eistentes na plataforma;
  • Os módulos "os" and "package" não são mais carregados por Lua, uma vêz que não podem mais ser utilizados. Devido a isso, o tamanho do código Lua ficou reduzido;
  • A documentação do projeto foi atualizada.

02 de Setembro de 2008

Foi liberada a versão 0.4! Segue abaixo o log das alterações:

  • Criado suporte para LPC2888 (preliminarmente);
  • Criado o módulo PWM;
  • Novos exemplos: TV-B-Gone (desliga a sua TV), piano (toca piano a partir do teclado de seu PC), pwmled (apaga/acende led), todas baseadas no novo módulo PWM;
  • Criado suporte para múltiplos espaços de memória (este recurso pode ser usado para obter vantagens tanto de memória RAM interna da CPU quanto de chips externos de memória RAM em placas com memória RAM);
  • Autorun: Caso o arquivo "autorun.lua" exista no sistema de arquivos, este é executado antes do inicio do shell;
  • Criado os módulos "pack" (compactação/descompactação de dados binários) e "bit" (operações binárias);
  • Atualizado o compilador do sistema, que agora está mais fácil de utilizar e sabendo como lidar com "placas", como também CPUs;
  • Foram modificados os módulos da plataforma existente para utilizar menos RAM e retornar uma mensagem de erro quando da tentativa de uso de um recurso não disponível no sistema;
  • A documentação do projeto foi atualizada.

28 de Julho de 2008

Recebi um relato sobre erros de linker após a execução de minhas intruções durante a compilação de um simples programa C++ para a CPU Cortex com compilador. Verifiquei e descobri que a biblioteca gcc's C++ (libstdc++) não estava gerando código corretamente para Cortex-M3. Minha culpa. Corrigi a página do tutorial. Foi necessário uma única alteração no passo 4, onde era preciso fornecer mais parametros para os comandos "make", e não somente o -CFLAGS. Obrigado pelo aviso.

07 de Julho de 2008

Estou ainda "polindo" os códigos fontes e criando mais documentação antes de atualizar a promeira versão par o CVS. Enquanto isso, Preparei uma ótima surpresa para todos voces que se mostraram interessados em eLua (e par voces que não se interessaram também, pois desejo muito que a partir de agora voces fiquem curiosos:) ). Logo, se você sempre quis iniciar seu PC com Lua, dê uma olhada aqui. É isso mesmo: sem OS, simplesmente o GRUB carrega um arquivo de boot tipo ELF! O arquivo tipo ELF é construido de acordo com a mesma estrutura em árvore que utilizei para montar dispositivos embarcados em eLua, precisei mudar somente a camada da plataforma (mais informações após a próxima atualização dos fontes e inclusão de novas documentações). Desejo que tenham gostado, da mesma maneira que eu. Enquanto isso é apenas uma prova de conceitos, não irei abandonar a idéia do "standalone Lua on PC", pois devem ter aplicações interesantes (pense em "BIOS scripting with Lua", Open Firmware com Lua ao invés de Forth, aplicações educacionais e muias outras.)
